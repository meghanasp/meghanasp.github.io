1. Arrays & Strings
  • Sorted? Use binary search or two pointers
  • Max/min or subarray? Try sliding window or DP
  • Duplicates? Use hash maps or sets
  • Fixed-size window? Use sliding window + two pointers
  • Min/max in each window? Use a deque (monotonic queue)
  • All subsets/permutations? Use backtracking
  • Bracket matching? Use stacks
2. Linked Lists
  • Cycle detection: slow & fast pointers (Floyd)
  • Reversing: Careful with prev, curr, next
  • Middle/intersection: Again, two-pointer pattern
3. Dynamic Programming
  • Look for overlapping subproblems and optimal choices
  • Knapsack/subset sums: 1D/2D DP arrays
  • String edits (LCS, edit dist.): DP matrix
4. Top K Elements
  • Top/smallest K items: Use a heap
  • Kth element: Try QuickSelect (average O(n))
5. Recursion Limits
  • Risk of stack overflow? Rewrite using an explicit stack or go iterative
6. Special Techniques
  • Sliding window: Great for variable subarrays
  • Monotonic stack: For next greater/smaller
  • Greedy: If local best ‚Üí global best
  • Tries: Perfect for prefix string problems
